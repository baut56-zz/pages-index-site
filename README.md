# pages-index-site

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面彼此独立，不针对任何特定域名或网站。仓库的主要目的是提供一个结构化的存储空间，方便管理和访问这些静态页面。

## 目录说明

仓库的目录结构如下：

```
pages-index-site/
├── README.md          # 本文件
├── index.html         # 仓库首页（可选）
├── pages/             # 存放所有归档 HTML 页面的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── assets/            # 公共资源目录（CSS、JS、图片等）
```

- `pages/`：所有独立 HTML 页面存放于此。文件名建议使用有意义的英文或数字标识，便于区分。
- `assets/`：存放页面共用的样式文件、脚本、图片等资源。如果页面需要独立资源，建议在页面所在目录下创建子目录管理。
- `index.html`：可选的入口页面，用于展示页面列表或导航。

## 页面归档说明

每个 HTML 页面应满足以下基本要求：

- 页面内容完整，包含必要的 HTML 结构（`<html>`、`<head>`、`<body>` 等）。
- 页面标题（`<title>`）清晰描述页面内容。
- 页面引用的资源（CSS、JS、图片等）尽量使用相对路径，确保在仓库内可正常访问。
- 页面之间没有强依赖关系，可单独打开使用。

## 维护说明

- 新增页面：将 HTML 文件放入 `pages/` 目录，并确保文件命名不与已有文件冲突。
- 更新页面：直接修改对应文件，建议在提交信息中简要说明修改内容。
- 删除页面：移除对应文件，并检查是否有其他页面引用该文件。
- 公共资源：如需修改 `assets/` 中的资源，请确认所有引用该资源的页面都能正常工作。
- 提交规范：建议使用简洁的提交信息，如“新增页面：page-xxx.html”或“更新页面：page-xxx.html”。

## 其他

本仓库仅作为静态页面归档使用，不提供任何形式的动态服务。页面内容由贡献者自行维护。
